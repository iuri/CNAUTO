Loading Data

This section defines json format when grid is loading data from server side.

Json format of data from grid to server side.


Here is a sample of Json string grid posts to server.

Notes


action: String. Alway be "load".
pageInfo: See Sigma.Grid.PageInfo object.
sortInfo: See Sigma.Grid.SortInfo object.
filterInfo: See Sigma.Grid.FilterInfo object.
parameters: Parameters user adds. See Sigma.Grid.addParameters

Json format of data from server side to grid.


Sample

Notes


data: Data content.
pageInfo: See Sigma.Grid.PageInfo object.
recordType: Type of data. Could be "array" or "object".

Saving Data

This section defines json format when grid is saveing data to server side.

Json format of data from grid to server side.


Sample

Notes


action: String. Alway be "save".
fieldsName: Array. Fields name.
insertedRecords : Array of array/object. Records inserted by user.
updatedRecords : Array of array/object. Records updated by user.
deletedRecords : Array of array/object. Records deleted by user.
recordType: Type of insertedRecords, updatedRecords, deletedRecords. Could be "array" or "object".
parameters: Parameters user adds. See Sigma.Grid.addParameters

Json format of data from server side to grid.


Sample

Notes


success: Bool. To specify whether CRUD operation is successful.
exception: Exception/error message.
